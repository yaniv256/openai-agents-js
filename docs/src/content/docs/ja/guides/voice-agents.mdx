## 概要

![リアルタイムエージェント](https://cdn.openai.com/API/docs/images/diagram-speech-to-speech.png)

音声エージェントを使うと、OpenAI の speech-to-speech モデル上に低レイテンシの音声インターフェイスを構築できます。SDK は Realtime API の考え方をそのまま保ちながら、元のイベントフローを `RealtimeAgent`、`RealtimeSession`、トランスポートヘルパーでラップし、ツール、ガードレール、ハンドオフ、セッション履歴を扱いやすくします。

内部では、公式の [WebRTC を用いた Realtime API](https://developers.openai.com/api/docs/guides/realtime-webrtc/)、[Realtime 会話](https://developers.openai.com/api/docs/guides/realtime-conversations/)、[音声活動検出](https://developers.openai.com/api/docs/guides/realtime-vad/) ガイドで説明されている同じ Realtime のコンセプトが引き続き適用されます。Voice Agents SDK は、その API の上に TypeScript ファーストのレイヤーを追加するため、トランスポートやイベント処理をゼロから作り直すのではなく、プロダクトロジックに集中できます。

<Aside type="note">
  ブラウザクライアントを WebRTC
  で接続する場合は、クイックスタートから始めてください。
  WebRTC、WebSocket、SIP、Twilio、Cloudflare Workers
  のどれを使うか検討している場合は、 まずトランスポートガイドに進んでください。
</Aside>

## SDK による追加要素

- ブラウザファーストの WebRTC セットアップと一時的なクライアントトークン
- サーバー側の WebSocket および SIP トランスポートオプション
- 自動的な割り込み処理とローカルの会話履歴更新
- リアルタイムハンドオフによるマルチエージェントオーケストレーション
- 関数ツール、ホスト型 MCP ツール、承認、委任パターン
- ライブ音声インタラクション向けの出力ガードレールとトレーシング対応

## Speech-to-speech の利点

Speech-to-speech モデルはユーザーの音声を直接処理するため、ターンごとに個別の音声認識、テキスト推論、音声合成のチェーンを構築する必要がありません。これによりレイテンシを抑えられ、リアルタイムアプリケーションでの割り込み、テキストと音声が混在する入力、ツール呼び出しがより自然に感じられます。
