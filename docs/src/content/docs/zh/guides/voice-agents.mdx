## 概述

![实时智能体](https://cdn.openai.com/API/docs/images/diagram-speech-to-speech.png)

语音智能体让你可以基于 OpenAI 语音到语音模型构建低延迟的语音交互界面。SDK 保留了 Realtime API 的心智模型，同时用 `RealtimeAgent`、`RealtimeSession` 和传输辅助工具封装原始事件流，让工具、护栏、交接和会话历史更易使用。

在底层，官方[基于 WebRTC 的 Realtime API](https://developers.openai.com/api/docs/guides/realtime-webrtc/)、[Realtime 对话](https://developers.openai.com/api/docs/guides/realtime-conversations/)和[语音活动检测](https://developers.openai.com/api/docs/guides/realtime-vad/)指南中的相同 Realtime 概念仍然适用。Voice Agents SDK 在该 API 之上添加了一个 TypeScript 优先层，让你可以专注于产品逻辑，而无需从零开始重建传输和事件处理。

<Aside type="note">
  如果你要使用 WebRTC 连接浏览器客户端，请从快速开始开始。 如果你正在
  WebRTC、WebSocket、SIP、Twilio 或 Cloudflare Workers 之间做选择，
  请先查看传输机制指南。
</Aside>

## SDK 新增内容

- 面向浏览器优先的 WebRTC 设置，支持临时客户端令牌。
- 服务器端 WebSocket 和 SIP 传输选项。
- 自动中断处理和本地对话历史更新。
- 通过实时交接实现多智能体编排。
- 函数工具、托管 MCP 工具、审批和委派模式。
- 面向实时语音交互的输出护栏和追踪支持。

## 语音到语音的优势

语音到语音模型会直接处理用户音频，因此你不必为每一轮对话构建单独的语音转文本、文本推理和文本转语音链路。这可以降低延迟，并让实时应用中的中断、文本与语音混合输入以及工具调用感觉自然得多。
