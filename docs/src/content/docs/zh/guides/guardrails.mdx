护栏可以与您的智能体并行运行，也可以阻塞执行直到它们完成，从而让您对用户输入或智能体输出执行检查和验证。例如，您可以在调用昂贵模型之前，先运行一个轻量级模型作为护栏。如果护栏检测到恶意使用，它可以触发错误并阻止高成本模型运行。

护栏有两种：

1. **输入护栏**会在初始用户输入上运行。
2. **输出护栏**会在最终智能体输出上运行。

## 工作流边界

护栏会附加到智能体上，但它们不一定会在工作流中的每个智能体上运行：

- **输入护栏**只会为链中的第一个智能体运行。
- **输出护栏**只会为生成最终输出的智能体运行。
- **工具护栏**会在每次函数工具调用时运行，其中输入护栏在执行前运行，输出护栏在执行后运行。

如果您需要在包含管理器或交接的工作流中围绕每个自定义函数工具调用执行检查，请使用**工具护栏**，而不是智能体级别的输入/输出护栏。

## 输入护栏

输入护栏按三个步骤运行：

1. 护栏会接收传递给智能体的相同输入。
2. 护栏函数执行并返回一个包装在 [`InputGuardrailResult`](/openai-agents-js/openai/agents/interfaces/inputguardrailresult) 中的 [`GuardrailFunctionOutput`](/openai-agents-js/openai/agents/interfaces/guardrailfunctionoutput)。
3. 如果 `tripwireTriggered` 为 `true`，则会抛出 [`InputGuardrailTripwireTriggered`](/openai-agents-js/openai/agents/classes/inputguardrailtripwiretriggered) 错误。

> **注意** 输入护栏用于用户输入，因此只有当该智能体是工作流中的*第一个*智能体时才会运行。护栏配置在智能体本身上，因为不同智能体通常需要不同的护栏。

### 执行模式

- `runInParallel: true`（默认）会让护栏与 LLM/工具调用并行启动。这样可以最大限度降低延迟，但如果护栏稍后触发，模型可能已经消耗了 token 或运行了工具。
- `runInParallel: false` 会在调用模型**之前**运行护栏，当护栏阻止请求时，可以避免 token 消耗和工具执行。当您更重视安全性和成本而非延迟时，请使用此模式。

## 输出护栏

输出护栏按 3 个步骤运行：

1. 护栏会接收智能体生成的输出。
2. 护栏函数执行并返回一个包装在 [`OutputGuardrailResult`](/openai-agents-js/openai/agents/interfaces/outputguardrailresult) 中的 [`GuardrailFunctionOutput`](/openai-agents-js/openai/agents/interfaces/guardrailfunctionoutput)。
3. 如果 `tripwireTriggered` 为 `true`，则会抛出 [`OutputGuardrailTripwireTriggered`](/openai-agents-js/openai/agents/classes/outputguardrailtripwiretriggered) 错误。

> **注意** 输出护栏只有在该智能体是工作流中的*最后一个*智能体时才会运行。对于实时语音交互，请参阅[构建语音智能体](/openai-agents-js/zh/guides/voice-agents/build#guardrails)。

输出护栏函数还会接收一个可选的 `details` 对象，其中包含底层 `modelResponse` 以及该轮生成的输出项。当仅凭最终输出不足以判断响应是否应通过时，可以使用它；例如，您想在触发护栏之前检查完整的生成项列表或提供方响应元数据。

## 工具护栏

工具护栏会包装**函数工具**，让您可以在执行前后验证或阻止工具调用。它们配置在工具本身上（通过 `tool()` 选项），并且会为该工具的每次调用运行。

在实践中，这意味着您会在自定义函数工具中，在 `tool({...})` 上设置 `inputGuardrails` 和/或 `outputGuardrails`。

- **输入工具护栏**会在工具执行前运行，并且可以用一条消息拒绝该调用，或抛出触发线。
- **输出工具护栏**会在工具执行后运行，并且可以用拒绝消息替换输出，或抛出触发线。

工具护栏会返回一个 `behavior`：

- `allow` — 继续执行下一个护栏或工具执行。
- `rejectContent` — 使用一条消息短路（跳过工具调用或替换输出）。
- `throwException` — 立即抛出触发线错误。

工具护栏适用于您使用 `tool()` 定义的函数工具。交接会以类似函数的工具形式呈现给模型，但它们会通过 SDK 的交接路径运行，而不是普通的函数工具管线，因此工具护栏不适用于交接调用本身。托管工具和内置执行工具（`computerTool`、`shellTool`、`applyPatchTool`）也不使用此护栏管线，并且 `agent.asTool()` 目前不直接暴露工具护栏选项。

## 触发线

当护栏失败时，它会通过触发线发出信号。一旦触发线被触发，runner 就会抛出相应错误并停止执行。
